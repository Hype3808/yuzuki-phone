【柚月小手机 - 核心新功能架构案】
项目代号：Real-World Meta-Universe (现实大厅联动系统)
📌 一、 核心概念与愿景
我们要在现有的“柚月小手机”插件中，引入一个颠覆性的概念：“线上（会话内）”与“线下（酒馆主大厅）”的二元分离与联动。
会话内（线上）：当用户点进某个具体的角色聊天窗口时，这属于**“剧情宇宙”**。小手机只读取当前聊天的上下文，时间线和社交关系都只针对当前这一个角色隔离。
酒馆主界面（线下/现实）：当用户没有点进任何聊天，而是停留在酒馆的主大厅（Lobby）时，这属于用户的“现实元宇宙”。
在这里，用户安装的所有角色卡，都变成了真实存在于用户“虚拟手机”通讯录里的联系人。
小手机要在后台静默运行，制造出一种“即使我不主动聊天，我的手机里也有一群活生生的人在过自己的生活”的错觉。
角色们会跨越会话的次元壁，在主大厅的背景下，互相在朋友圈点赞、在微博互怼，甚至主动给用户发来微信弹窗。
⚙️ 二、 核心功能模块设计
为了实现这个愿景，我们需要开发以下几个核心模块。请 AI 助手先理解整体架构，不要急于写代码。
目标：精准判断用户当前处于“会话中”还是“大厅中”。
机制：
监听酒馆的 CHAT_CHANGED 等事件。
判断条件：当 SillyTavern.getContext().chatId 不存在、或者当前 DOM 没有激活的聊天窗口时，判定为进入 Lobby 模式。
当处于 Lobby 模式时，开启一个后台的心跳/定时器（例如每隔 3-5 分钟触发一次“现实事件生成检测”）。
目标：在不依赖任何单一聊天记录的情况下，提取构建“现实元宇宙”所需的所有基础数据。
需要提取的数据：
全局用户信息：读取 name1（用户名）以及 #persona_description（用户当前的人设说明）。
全局角色通讯录：遍历酒馆的 window.characters 数组。提取所有安装的角色卡的 name（名字）、avatar（头像）、personality（性格）。为了防止 Token 溢出，description（详细设定）只截取前 150 个字符作为“人物印象标签”。
世界书注入：复用现有的 WorldbookManager，读取全局激活的背景设定（如这是现代都市、赛博朋克还是魔法世界）。
目标：在大厅模式下，通过手机独立的 API 发起极低优先级的后台请求，生成随机事件。
机制：
使用我们现有的 ApiManager 进行请求，绝对不能污染任何角色的本地聊天存档。
Prompt 组装策略：
System: 你是一个真实手机的后台数据引擎。当前用户（设定：{用户信息}）正看着手机屏幕。他的手机通讯录里有以下联系人：{角色卡列表摘要}。当前世界背景为：{世界书}。
User: 请根据目前的时间（早上/下午/深夜），以及上述角色们的人设性格，随机抽取 1-3 个角色，生成一个“此刻正在发生的手机日常事件”。可以是：A给用户发了条闲聊微信；B发了条微博；C在朋友圈发了张照片，D在下面吐槽。
严格限制输出格式，要求 AI 按照我们既定的 <wechat>、<Weibo>、<Moments> 等标签输出。
目标：把大模型在后台编好的“戏”，变成手机屏幕上的真实通知。
机制：
当 Meta-API 返回数据后，交由我们现有的 tag-filter.js 和解析正则进行清洗。
数据分发：
如果是 <wechat> 标签：将其写入 GlobalSocialStore 或对应的 WechatData 的本地数据库（作为未读消息存入），并增加微信图标的红点，触发微信来消息的顶部弹窗（Notification）。
如果是 <Weibo> 标签：写入推荐流数据库，触发微博更新通知。
关键点：即使用户在酒馆大厅发呆，小手机右上角也会突然亮起红点，下拉通知栏会看到：“张三给你发来一条微信”，用户点开手机壳，打开微信，就能看到刚才 AI 后台生成的跨次元对话！
🛡️ 三、 实施原则与代码边界（AI 必须遵守）
绝对隔离原则：在 Lobby 模式下生成的任何聊天内容，绝对不可以通过酒馆原生的 API 发送，也绝对不可以写入任何酒馆原生的 .jsonl 聊天记录文件中。所有现实联动产生的数据，必须全部且仅存入小手机自己的 storage.js（例如 wechat_data 的 localStorage 键）中！
Token 防护机制：如果用户安装了 100 个角色卡，把所有设定发给大模型会导致 API 崩溃或天价账单。所以 Global Data Extractor 必须具备**“随机抽选机制”或“活跃度排序机制”**。每次后台触发时，最多只挑 5-8 个角色送给大模型进行排戏。
静默与无感：大厅事件生成必须是静默的。如果没有生成成功，或者 API 报错，直接默默吞掉错误（Catch error），绝不能在界面上弹出红色报错打断用户的浏览体验。
